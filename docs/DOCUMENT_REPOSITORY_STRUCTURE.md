## Implementation

### Database Schema

```sql
CREATE TABLE agent_document_library (
  id TEXT PRIMARY KEY DEFAULT gen_random_uuid()::text,
  agent_id TEXT NOT NULL,
  category TEXT NOT NULL, -- 'regulatory', 'framework', 'template', 'industry'
  country_code TEXT, -- 'US', 'EU', 'UK', 'APAC', 'INTL'
  industry TEXT, -- 'financial', 'healthcare', 'technology', etc.
  standard_name TEXT, -- 'PMBOK', 'ISO 31000', 'GDPR', etc.
  document_title TEXT NOT NULL,
  document_path TEXT NOT NULL, -- 'docs://governance/SOX_Compliance_Checklist.pdf'
  file_type TEXT, -- 'pdf', 'docx', 'xlsx', 'md'
  version TEXT,
  description TEXT,
  tags TEXT[], -- ['compliance', 'sox', 'financial']
  applicable_phases TEXT[], -- ['initiation', 'planning', 'execution']
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP DEFAULT NOW()
);

CREATE INDEX idx_agent_docs_agent ON agent_document_library(agent_id, category);
CREATE INDEX idx_agent_docs_country ON agent_document_library(country_code);
CREATE INDEX idx_agent_docs_industry ON agent_document_library(industry);
CREATE INDEX idx_agent_docs_standard ON agent_document_library(standard_name);
```

### API Endpoints

```bash
# Get documents for agent
GET /api/agents/:agentId/documents
Query: ?category=regulatory&country=US&industry=financial

# Search documents
GET /api/admin/documents/search
Query: ?q=GDPR&agent=governance

# Upload document
POST /api/admin/documents
{
  "agentId": "governance",
  "category": "regulatory",
  "countryCode": "EU",
  "standardName": "GDPR",
  "title": "GDPR Compliance Checklist",
  "file": <multipart upload>
}

# Get document by path
GET /api/documents/governance/SOX_Compliance_Checklist.pdf
```

### Camunda Integration

Documents are automatically attached when DMN rules fire:

```xml
<outputEntry>
  <text>"docs://governance/SOX_Compliance_Checklist.pdf,docs://governance/ISO21500_Standard.pdf"</text>
</outputEntry>
```

The system parses this output and:
1. Fetches documents from repository
2. Attaches to agent context
3. Makes available in LLM prompt

## Auto-Population Script

```bash
npm run seed:documents
```

This will:
1. Create all folder structures
2. Generate placeholder PDFs for each document
3. Populate database with metadata
4. Link documents to agents
5. Tag documents with relevant keywords

## Usage in Agents

### Example: Governance Agent

```typescript
// Agent detects compliance violation
const decision = await camunda.evaluateAgentCollaboration({
  sourceAgent: 'governance',
  complianceViolations: 3,
  severity: 'high'
});

// Decision output includes documents
decision.requiredDocuments // ["docs://governance/SOX_Compliance_Checklist.pdf", ...]

// Fetch documents
const docs = await documentRepository.getDocuments(decision.requiredDocuments);

// Attach to agent context
const systemPrompt = `
You are the Governance Agent.

Relevant compliance documents:
1. SOX Compliance Checklist (2023) - docs://governance/SOX_Compliance_Checklist.pdf
2. ISO 21500 Standard - docs://governance/ISO21500_Standard.pdf

Based on these standards, assess the compliance violation...
`;
```
