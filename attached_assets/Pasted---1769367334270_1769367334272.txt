  "Also integrating it with the attribute creation"                                                                  
                                                                                                                     
  Answer: /server/lib/PolicyExtractionService.ts:264-295 (approvePolicy method)                                      
                                                                                                                     
  When a policy is approved by a human:                                                                              
  1. Custom attributes are auto-created from the extracted policy requirements                                       
  2. Each attribute is linked back to the policy via sourcePolicyId                                                  
  3. Attributes are marked as autoGenerated: true

  Example: If policy says "All projects must track team morale", the system creates a teamMorale custom attribute    
  that the TMO agent can use.                                                                                        
                                                                                                                     
  ---                                                                                                                
  "And interface with the rules builder"                                                                             
                                                                                                                     
  Answer: /server/lib/PolicyExtractionService.ts:297-320                                                             
                                                                                                                     
  When a policy is approved:                                                                                         
  1. Collaboration rules are auto-created from policy requirements                                                   
  2. Rules use json-rules-engine format (conditions + actions)                                                       
  3. Each rule is linked via sourcePolicyId                                                                          
  4. Marked as mandatory: true (compliance rules cannot be disabled)                                                 
  5. Executed by existing AgentCollaborationRulesEngine.ts                                                           
                                                                                                                     
  Example: If policy says "Budgets over $500K require CFO approval", the system generates a rule that automatically  
  triggers when the condition is met.
